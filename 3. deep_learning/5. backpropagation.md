back propagation is a method used to update the parameters (weights and biases) of the neural network

## Chain Rule Engine
this update is determined by the **gradient**.

mathematically the gradients *direction* refers to the steepest direction or the greatest rate of increase and the *magnitude* refers to the rate at which the function increases. so if we want to minimize the error we would update the params in the opposite direction to minimize the error.

In a simple single-neuron model, we could apply gradient descent directly. However, in a deep, multi-layered architecture, running gradient descent directly would only update the final output layer. To train the hidden layers underneath, we must measure exactly how much each individual node affects our final prediction, and decide how much it needs to change based on that influence. 

To achieve this, we use **partial derivatives**. The notation $\frac{\partial y}{\partial x}$ can be intuitively described as: "how much $x$ affects $y$." This is exactly the tool we need to map out our network's inner relationships.

If we look back at our core forward pass equations:

$$
Z = X \cdot W + b
$$

$$
A = \sigma(Z)
$$

$$
J = \frac{1}{m} \sum_{j=1}^{m} \sum_{\text{neurons}} (y^j_i - a_i^j)^2
$$

We notice that while the total cost ($J$) is definitely affected by any given weight matrix ($W$), it is not affected by it *directly*. Instead, a change in weights ripples through a sequence of downstream dependencies:
1. Changing the weights ($W$) directly changes the pre-activation scores ($Z$) $\rightarrow \frac{\partial Z}{\partial W}$
2. Changing those scores ($Z$) changes the non-linear activation outputs ($A$) $\rightarrow \frac{\partial A}{\partial Z}$
3. Changing those activations ($A$) changes the final calculated cost ($J$) $\rightarrow \frac{\partial J}{\partial A}$

Because this structural link is indirect, we can multiply these local rates of change together to find the global impact. This mathematical domino effect is known as the **Chain Rule**:

$$
\frac{\partial J}{\partial W^L} = \frac{\partial J}{\partial A^L} \cdot \frac{\partial A^L}{\partial Z^L} \cdot \frac{\partial Z^L}{\partial W^L}
$$

### Propagating Deeper into Previous Layers

As we travel backward to earlier layers in the network, we simply stack additional partial derivative terms onto the chain to bridge the layers together. For example, to find how much the weights of the hidden layer ($W^{L-1}$) affect our total cost, we pass the error signal through the intermediate activations ($A^{L-1}$):

$$
\frac{\partial J}{\partial W^{L-1}} = \frac{\partial J}{\partial A^L} \cdot \frac{\partial A^L}{\partial Z^L} \cdot \frac{\partial Z^L}{\partial A^{L-1}} \cdot \frac{\partial A^{L-1}}{\partial Z^{L-1}} \cdot \frac{\partial Z^{L-1}}{\partial W^{L-1}}
$$

By tracking these derivative connections in reverse, every single parameter across our entire network learns exactly how much responsibility it bears for the final error.

## Math: Applying this mathematically
#### part 1: The Cost Derivative ($\frac{\partial J}{\partial A^L}$)

We begin at the very end of the network by checking the error of our final predictions. Our Mean Squared Error cost function across all samples and output neurons is defined as:

$$
J = \frac{1}{m} \sum_{j=1}^{m} \sum_{i=1}^{h} (y^j_i - a_i^j)^2
$$

Taking the partial derivative with respect to our activation prediction ($a_i^j$) requires the power rule combined with the inner chain rule. Because the activation term is negative inside the parentheses ($-a_i^j$), its inner derivative yields a $-1$, which flips the subtraction order:

$$
\frac{\partial J}{\partial a_i^j} = \frac{2}{m} (a_i^j - y^j_i)
$$

#### Vectorization:
When vectorizing this expression, tracking individual summation symbols ($\sum$) no longer makes sense because we handle our dataset samples and processing nodes entirely as unified matrices. Eliminating the scalar loops allows us to calculate the global error gradient across the entire batch simultaneously via clean element-wise matrix subtraction:

$$
dA^L = \frac{\partial J}{\partial A^L} = \frac{2}{m} (A^L - Y)
$$

Where both $A^L$ and $Y$ are matrices of shape $(m \times h)$.

##### in code:

```python
# Derivative of cost with respect to final activations 
dAL = (2 / m) * (A_final - Y) # Shape: (m x h)
```

#### part 2: activation derivative ($\frac{\partial A^L}{\partial Z^L}$)

##### 1. Sigmoid

$$
\sigma(Z) = \frac{1}{1+e^{-Z}}
$$

the derivative of the sigmoid function is quite simple.

$$\sigma'(Z) = A \cdot (1 - A)$$

##### 2. ReLU

$$g(z) = \begin{cases} z & \text{if } Z > 0 \\ 0 & \text{if } Z \le 0 \end{cases}$$

the ReLU is a piecewise function so its derivative is binary.

$$g'(Z) = \begin{cases} 1 & \text{if } Z > 0 \\ 0 & \text{if } Z \le 0 \end{cases}$$

there is no vectorization needed here so lets continue 

so the global chain formula becomes:

$$dZ = \frac{\partial J}{\partial Z} = \frac{\partial A^L}{\partial Z^L} \cdot \frac{\partial J}{\partial A^L}$$

this simply means how the $J$ changes with respect to $Z$.

##### in code: 

```python
# For Sigmoid layers: 
dZ = dAL * (A * (1 - A)) 

# For ReLU layers: 
dZ = dAL * (Z > 0) # (Z > 0) creates a matrix of True/False (1/0) automatically!
# equals an if statement with if Z > 0 return 1 elif return zero
```

#### part 3: linear hypothesis derivative ($\frac{\partial Z^L}{\partial W^L}$, $\frac{\partial Z^L}{\partial b^L}$)

We now look at the linear combination step for a single layer:

$$
Z^L = A^{L-1} \cdot W^L + b^L
$$

##### 1. The Bias Gradient ($db^L$)
The derivative of $Z^L$ with respect to $b^L$ is simply $1$. 

By applying the chain rule, the local gradient is multiplied by the downstream error signal ($dZ^L$). Since the bias affects all samples across the entire batch, we sum the gradients across the batch dimension ($m$):

$$
db^L = \frac{\partial J}{\partial b^L} = \sum_{j=1}^m dZ^{L,(j)}
$$

##### 2. The Weight Gradient ($dW^L$)
The derivative of $Z^L$ with respect to $W^L$ is the incoming activation matrix from the previous layer, $A^{L-1}$.

$$\frac{\partial Z^L}{\partial W^L} = A^{L-1}$$

When vectorizing this across the entire batch, matrix multiplication naturally handles both the summation over all $m$ samples and the dimension matching. Because matrix multiplication is order-dependent, we transpose $A^{L-1}$ to align the features:

$$
dW^L = \frac{\partial J}{\partial W^L} = (A^{L-1})^T \cdot dZ^L
$$

##### in code:

```python
# Calculate gradients for weights and biases
dW = np.dot(A_prev.T, dZ) 
# dZ shape: (m x h)
# A_prev shape: (m x n) -> A_prev.T shape: (n x m)
# Resulting dW shape: (n x h), matching the weight matrix dimensions

db = np.sum(dZ, axis=0, keepdims=True) 
# Sums across the batch rows, collapsing (m x h) down to (1 x h)
```

---

#### part 4: Building the Bridge ($\frac{\partial Z^L}{\partial A^{L-1}}$)

now we want to calculate how much the previous layers activations ($A^{L-1}$) affect this layer's linear score ($Z$).

$$Z^L = A^{L-1} \cdot W^L + b^L$$

the derivative of it with respect to $A^{L-1}$:

$$\frac{\partial Z^L}{\partial A^{L-1}} = W^L$$

the incoming gradient for layer L-1:

$$\mathrm{dA}_{\mathrm{prev}} = \underbrace{dZ^L}_{m \times h} \cdot \underbrace{(W^L)^T}_{h\times n}$$

the weight is transposed to match the inner layer of the 2 matrices.

##### in code:

```python
dA_prev = np.dot(dZ, layer.W.T)
```
